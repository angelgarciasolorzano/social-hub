# Plantilla de Revisión de PR

Plantilla con la estructura recomendada para **enviar reviews** consistentes sobre Pull Requests.

## Ubicación

`.github/REVIEW_TEMPLATE/review_template.md`

## Cuándo se usa

A diferencia de la [plantilla de PR](pull_request_template.md), GitHub **no** la carga automáticamente. Es una referencia que tú (o tu equipo) usan para redactar reviews con el mismo formato cada vez.

## Estructura

La plantilla tiene cinco secciones:

| Sección | Propósito |
|---|---|
| 📋 Review Summary | Resumen ejecutivo: qué revisaste y tu impresión general. |
| ✅ What looks good | Aspectos positivos: arquitectura, claridad, cobertura de tests, etc. |
| ⚠️ Requested changes | Cambios **bloqueantes** antes de aprobar (con checkboxes). |
| 💡 Suggestions | Mejoras opcionales que no bloquean la aprobación. |
| 🧪 Notes | Lo que verificaste localmente, edge cases revisados, comentarios extra. |

## Cómo usarla

Abre `.github/REVIEW_TEMPLATE/review_template.md`, copia el contenido y pégalo en el cuerpo del review de GitHub.

## Cómo personalizarla

Edita `.github/REVIEW_TEMPLATE/review_template.md` directamente. Los cambios aplican para futuros usos — no requiere sincronización.
